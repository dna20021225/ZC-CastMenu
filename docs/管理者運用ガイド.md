# 管理者運用ガイド

ZC-CastMenu の管理者アカウント運用とパスワードリカバリ手順をまとめます。

## 認証の仕組み

- 管理画面のログインは NextAuth.js + Credentials（ユーザー名 + パスワード）で行います。
- ユーザー情報は SQLite の `admins` テーブルに保存され、パスワードは `bcrypt` でハッシュ化されています。
- 初期管理者:
  - ユーザー名: `admin`
  - 初期パスワード: `password1234`
  - 初回ログイン後、必ず管理画面の「パスワード変更」から変更してください。

### レガシーフォールバックについて

DB に `admins` レコードが無い／DB アクセスに失敗した場合のみ、`admin / password1234` の固定資格情報でログインできます。これは移行期の保険であり、本番運用が安定したら以下の環境変数で無効化できます。

```env
DISABLE_LEGACY_LOGIN=1
```

### `DISABLE_LEGACY_LOGIN=1` を本番に反映する手順

「DB ログイン経路が安定して動いている」状態が確認できてから以下を実施してください。

#### 事前確認チェックリスト

下記すべてに ✅ が付いてから無効化してください。**1つでも未確認だと、自分が管理画面から締め出されます。**

1. ✅ Vercel の Production 環境変数で `SETUP_SECRET` が設定されている
   - `Vercel > Project > Settings > Environment Variables` で `SETUP_SECRET` の存在を確認
   - 値そのものはダッシュボードでは確認不可。手元の安全な場所（パスワードマネージャ等）に控えがあるか確認
2. ✅ 本番 `/api/init-db` を最低 1 回 POST して `admins` テーブルにシード済みである
   ```bash
   curl -X POST https://zc-castmenu.vercel.app/api/init-db
   ```
3. ✅ 上記シード後に **DB の `admin` アカウントで実際にログインできる**ことを本番で確認した
   - 「ログインできた = DB 経路が生きている」を意味します。これが取れていない状態でレガシーを切ると、両経路ともダメな状態に陥り得ます。
4. ✅ 初期パスワード `password1234` から **既に別パスワードに変更済み**である
   - 変更後ログインまで通っていれば DB の `password_hash` が正常に更新されているとみなせます。

#### 反映手順

1. Vercel ダッシュボードを開く
2. `Project > Settings > Environment Variables`
3. `Add New` で以下を追加
   - Key: `DISABLE_LEGACY_LOGIN`
   - Value: `1`
   - Environments: **Production**（必要に応じて Preview も）
4. 既存の本番デプロイメントを再デプロイ（環境変数追加だけではランタイムに反映されないため）
   - `Deployments` タブ → 最新の Production デプロイの `...` メニュー → `Redeploy`
   - `Use existing Build Cache` のチェックは外しても付けてもよい

#### 反映後の確認

1. ✅ DB の管理者アカウント＋現在のパスワードでログインできる
2. ✅ レガシー資格情報（`admin / password1234`）ではログインが**拒否される**こと
   - 「ログイン情報が正しくありません」相当のエラーが返れば成功

#### 万が一締め出された場合のリカバリ

`SETUP_SECRET` が手元にあれば、本ドキュメント下部の「パスワードを忘れた場合のリカバリ」手順でパスワードを再設定できます。`DISABLE_LEGACY_LOGIN` の値は `SETUP_SECRET` 経由のリカバリ動作には影響しません。

## パスワード変更（通常運用）

1. 管理画面にログイン
2. ダッシュボード →「システム管理」→「自分のパスワードを変更」
3. 現在のパスワードと新しいパスワード（8 文字以上）を入力して送信

## パスワードを忘れた場合のリカバリ

管理者がログイン情報を忘れた場合は、`SETUP_SECRET` を使って強制的にパスワードをリセットします。

### 前提: 環境変数 `SETUP_SECRET` の設定

Vercel のプロジェクト設定で以下の環境変数を設定しておきます（本番デプロイ前に必須）。

```env
SETUP_SECRET=<32文字以上のランダム文字列>
```

生成例:

```bash
openssl rand -hex 32
```

この値は **オーナー（あなた）だけが知っている前提** で運用してください。漏洩したら直ちに再生成してください。

### リカバリ手順

ターミナルから以下を実行します。

```bash
curl -X POST https://zc-castmenu.vercel.app/api/admin/setup \
  -H "Content-Type: application/json" \
  -H "x-setup-secret: <SETUP_SECRET の値>" \
  -d '{"username":"admin","newPassword":"<新しいパスワード>"}'
```

成功すると以下のレスポンスが返ります。

```json
{ "message": "パスワードをリセットしました", "username": "admin" }
```

その後、新しいパスワードでログインできます。

### 注意点

- `username` は admin 以外を指定すれば、そのユーザー名のレコードが新規作成されます（多店舗展開時の管理者追加にも使えます）。
- `newPassword` は 8 文字以上必須です。
- `SETUP_SECRET` が未設定の場合、API は `503` を返します。Vercel の環境変数設定を確認してください。
- 旧 `GET /api/admin/setup` 経路は廃止されました（誰でも叩けて admin123 にリセットされる重大なセキュリティホールだったため）。

## 多店舗展開時の管理者追加

店舗ごとに異なる管理者を持たせる場合、以下のいずれかで追加します。

### A. 管理画面から追加（推奨）

1. ログイン後「ダッシュボード」→「システム管理」→「管理者アカウント一覧・追加」
2. 「新規管理者追加」をクリック
3. ユーザー名・メール・パスワードを入力して作成

### B. SETUP_SECRET 経由で追加

```bash
curl -X POST https://<店舗ドメイン>/api/admin/setup \
  -H "Content-Type: application/json" \
  -H "x-setup-secret: <SETUP_SECRET>" \
  -d '{"username":"shop2-admin","newPassword":"<初期パスワード>"}'
```
