### 仕様

- キャスト詳細画面 `/cast/[id]` で画面全体の横スワイプを検知
  - 左スワイプ → 次のキャスト
  - 右スワイプ → 前のキャスト
- 並び順は 007 で実装した `display_order` に従う（ホーム画面と同順）
- 端でループする（最後 → 最初 / 最初 → 最後）
- 写真エリア（`data-photo-carousel="true"`）内のスワイプは写真カルーセル優先のため無視

### 変更内容

1. **新規カスタムフック** `src/hooks/useHorizontalSwipe.ts`
   - TouchEvent ベースの横スワイプ検知
   - 縦移動が大きい場合（縦スクロール）は無視
   - `ignoreInsideSelector` で除外領域を指定可能
2. **キャスト詳細ページ** `src/app/cast/[id]/page.tsx`
   - 一覧API（`/api/casts?limit=100`）を別途取得し、display_order 順の ID 配列を保持
   - 写真エリアに `data-photo-carousel="true"` 属性を付与

### 追加改修（2026-06-01）

#### シームレス遷移化

- 当初は `router.push` でページ遷移していたため、スワイプ毎に画面がリロードされていた
- 改修後：
  - 表示中のキャストを内部 state (`currentCastId`) で管理
  - 取得済みキャストを Map<id, CastDetail> でクライアントキャッシュ
  - 隣接2件を `useEffect` でバックグラウンドプリフェッチ
  - 進行中の fetch は Promise キャッシュで共有（React StrictMode の二重実行にも対応）
  - スワイプ時はキャッシュ参照→即座に state 差し替え、URL は `window.history.replaceState` で静かに更新
  - リロードしても表示中のキャストが出る、PWA らしい体験

#### 写真表示のフォールバック

- 現状のDBデータでは `cast_photos` テーブルが空のキャストが多数（avatar_url のみ設定）
- 詳細画面で `cast.photos` が空のとき `avatar_url` を1枚カルーセルとして表示
- 編集API (`PUT /api/casts/[id]`) で photos 配列が空でも、avatar_url が指定されていれば `cast_photos` に1枚保証する処理を追加（抜本対応）
