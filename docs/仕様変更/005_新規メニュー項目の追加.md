### 仕様

- 既存の `/menu/pricing`（準備中プレースホルダー）を「**料金メニュー**」ページに転用
- ドリンクの仕組み（drink_categories / drinks）を流用して **menu_categories / menu_items** テーブルを追加
- 管理画面でカテゴリ・項目のCRUDが可能

### 変更内容

1. **DBスキーマ**: `menu_categories` / `menu_items` テーブル追加（drinks の構造そのまま）
2. **マイグレーション** (`/api/init-db`): 初期データ投入（カテゴリ・項目が空のときのみ）
3. **API**: `GET/POST/PUT/DELETE /api/menu`（drinks API と同じパターン、type: 'category' | 'item'）
4. **顧客向け** `/menu/pricing`: 準備中プレースホルダー撤去、カテゴリ別の一覧表示
5. **管理画面** `/admin/pricing`: 既存の「未実装」表示を撤去、ドリンク管理と同等のCRUD UI

### 本番デプロイ後の作業

`POST /api/init-db` を1度叩く（テーブル作成 + 初期データ投入）
