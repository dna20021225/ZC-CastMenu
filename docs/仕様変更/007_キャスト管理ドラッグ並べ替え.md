### 仕様

- 管理画面 `/admin/casts` の各行に `≡` ハンドル（GripVerticalアイコン）を追加し、ドラッグで並び替え可能
- 並び替えた結果は即座にAPIへ送信、楽観的UI更新（失敗時は元に戻す）
- バッジ優先ソートは撤廃し、すべての画面で `display_order` 順に統一
  - No.1/No.2/No.3 バッジはラベル・デザイン上の意味だけが残り、表示順は手動順
  - 「No.1キャストを一番上に」したい場合は管理画面で手動で並べる

### 変更内容

1. **DBスキーマ**: `casts.display_order INTEGER DEFAULT 0` カラム追加
2. **マイグレーション** (`/api/init-db`): 既存DBに対し ALTER TABLE で追加 + 作成日順で初期化
3. **新規API**: `PATCH /api/casts/reorder` — body の `ids: string[]` で渡された順序で `display_order` を更新
4. **キャスト作成API**: 新規登録時 `display_order` に末尾の値を自動付与
5. **キャスト一覧API**: ORDER BY を `display_order ASC, created_at ASC` に変更（バッジ優先廃止）
6. **管理画面UI**: `@dnd-kit/sortable` 導入。`≡` ドラッグハンドルでテーブル並び替え対応
