## 現状の問題・要望

プロフィール入力欄に2つの問題がある。

> プロフィール入力欄の修正
> →既存の数値を消去した際に0表記されないようにしてほしい
> →非公開または空白でも更新可能にしてほしい（年齢、身長、血液）

1. 数値入力欄で既存の値を消去すると「0」が表示されてしまう
2. 年齢・身長・血液型が空白や非公開の状態では更新（保存）ができない

## 期待する動作・変更内容

1. 数値入力欄の値を消去した際、「0」ではなく空欄の状態を維持する
2. 年齢・身長・血液型を空白または「非公開」のまま保存・更新できるようにする

## 対象画面・機能

- 管理画面のキャストプロフィール編集画面
- 対象フィールド：年齢、身長、血液型

### 変更内容

1. **数値入力欄の0表記を解消**
   - 編集/新規作成画面の年齢・身長フィールドで、`parseInt(value) \|\| 0` の罠（空文字列を0に変換）を解消
   - state を `number \| null` 型に変更し、空欄なら `null` を保持
   - 表示時は `value={formData.age ?? ""}` で空欄を維持

2. **空欄/非公開での保存対応**
   - 年齢・身長・血液型を空欄または「非公開」のまま保存・更新可能に
   - フォームの `required` 属性を撤去
   - 血液型のselectに `非公開`（value=""）オプションを追加

3. **血液型のDB保存**
   - 既存では `blood_type` が DB に保存されていなかった形骸化フィールドだったため、DBスキーマに `blood_type` カラムを追加
   - キャスト詳細画面・一覧画面で空欄なら「非公開」と表示

4. **DBマイグレーション**
   - `casts` テーブルの `age`/`height` の `NOT NULL` 制約を撤去（既存DBはテーブル再構築で対応）
   - `/api/init-db` 経由で既存本番DB（Turso）に対しても適用可能

### 動作確認（Playwright E2E）

- ✓ 編集画面で年齢を空欄にしたとき `value === ""` を維持（0表記されない）
- ✓ 編集画面で身長を空欄にしたとき `value === ""` を維持
- ✓ 年齢・身長・血液型すべて空欄/非公開のまま保存可能
- ✓ 保存後、一覧・詳細画面で「非公開」表示を確認
- ✓ 新規作成画面の初期値がすべて空欄
- ✓ 新規作成画面で名前のみ入力して保存成功
